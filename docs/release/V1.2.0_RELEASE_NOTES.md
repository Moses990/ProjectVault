# Project Vault V1.2.0 Release Notes

**Release Date:** 2026-06-26

## Overview

V1.2.0 completes all strategic features from the code review action plan, adding file preview capabilities, directory tree navigation, AI-powered project analysis, CSV export, real-time scanner status tracking, structured logging, WebView2 runtime detection, and a frontend test suite.

## New Features

### File Preview & Thumbnails (P7)
- Server-side thumbnail generation using Pillow (JPEG/PNG/GIF/WebP/BMP/TIFF/SVG)
- Thumbnails cached to tempdir with configurable size (32-800px)
- Text file preview with 64KB limit and multi-encoding support (UTF-8/GBK/Latin-1)
- Frontend `FilePreview` modal component supporting image, PDF, and text preview

### Directory Tree Navigation (P9)
- Recursive directory tree built from database `relative_dir` GROUP BY query
- Expand/collapse nodes with file counts
- Left-right layout in Files tab: 220px sidebar + file list
- Breadcrumb navigation with clickable path segments
- Directory filtering via selected path state

### AI Project Analysis (P5)
- Real AI provider connectivity test (HTTP GET `{base_url}/models` + Bearer auth)
- `POST /{project_id}/ai-analyze` endpoint calls first enabled provider's `/chat/completions`
- Chinese-language prompt for architecture/interior design analysis
- Results stored in `ai_metadata` table with structured JSON
- Frontend `AiTab` with analysis button, loading state, and error display

### CSV Export (P12)
- `GET /{project_id}/files/export` returns CSV via StreamingResponse
- `GET /projects/{project_id}/drawings/export` returns drawing inventory CSV
- Frontend export buttons trigger browser download via `<a download>`

### Scanner Status Tracking (B4)
- `ScannerState` singleton tracks real watcher/processor state
- Status updates during scan: IDLE → SCANNING → IDLE
- Queue length, current project, and total events processed exposed via API
- `GET /api/v1/scanner/status` returns live state instead of hardcoded stub

### Settings Enhancements
- Auto-scan toggle in Settings page (enables/disables file watcher)
- Configurable backup retention count (1-100, default 10)
- Backup retention UI with number input and description

### Dashboard Favorites (P10)
- Favorites section on Dashboard between metrics and recent projects
- Star icon and card grid layout for favorited projects

### Page Metadata (F11)
- `<title>Project Vault</title>` and meta description added to layout

## Improvements

### Backend Security & Validation
- Query length validation (200 char limit) on search and project endpoints (B6)
- `MaintenanceRequest.now` validation with max 50 chars (B7)
- `os.startfile` path resolution uses `path.resolve()` for security (B8)
- Backup retention configurable via settings instead of hardcoded (B9)
- Rate limiting middleware: 100 requests/minute per IP, returns 429 (B10)

### Rust Desktop (D5, D7)
- Structured logging via `tracing` + `tracing-subscriber` with env filter
- Log points at key lifecycle events: startup, port allocation, backend spawn, health check, window creation, exit
- WebView2 runtime detection before app startup (registry + filesystem check)
- Clear error message if WebView2 runtime is missing

### Frontend Testing (F12)
- Vitest + Testing Library + jsdom test suite
- `api.test.ts`: backendPort utility tests
- `DirectoryTree.test.tsx`: component rendering, selection, click handling
- `npm test` and `npm run test:watch` scripts added

## Bug Fixes

- `settings_put` now persists `auto_scan` value (was missing from saved values dict)
- `saveSettings` call includes `backup_retention` field to match Settings type

## Technical Details

### Dependencies Added
- **Backend:** `Pillow>=10.0` (thumbnail generation)
- **Frontend (dev):** `vitest@^3.2.1`, `@testing-library/react@^16.3.0`, `@testing-library/jest-dom@^6.6.3`, `jsdom@^26.1.0`
- **Desktop (Rust):** `tracing@0.1`, `tracing-subscriber@0.3` (env-filter feature)

### API Changes
- `GET /api/v1/assets/{file_id}/thumbnail?size=200` — thumbnail image
- `GET /api/v1/assets/{file_id}/text` — text file content (64KB limit)
- `GET /api/v1/projects/{project_id}/file-tree` — directory tree structure
- `GET /api/v1/projects/{project_id}/files/export` — CSV export
- `GET /api/v1/projects/{project_id}/drawings/export` — CSV export
- `POST /api/v1/projects/{project_id}/ai-analyze` — trigger AI analysis
- `GET /api/v1/projects/favorites` — favorited projects list

### Database
- No schema changes required
- `ai_metadata` table reused for AI analysis results (INSERT OR REPLACE)

## Upgrade Path

1. Install V1.2.0 NSIS installer (replaces V1.0.0/V1.1.0)
2. Database automatically migrated (no manual steps)
3. Settings preserved (root_path, scan_interval, theme, backup_retention)
4. AI providers require re-testing if base_url changed

## Known Limitations

- Sandbox FUSE mount doesn't support opendir/file writes; `next build` and `cargo check` must run on host
- `next dev` mode ChunkLoadError is a known issue with `output:export` + `trailingSlash:true`; production build unaffected
- Grep/Glob tools unavailable in sandbox (rg.exe ENOENT); use bash grep/find instead
